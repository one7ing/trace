# Trace 知识库 & AI 记忆架构设计

---

## 一、知识库功能

### 1. 架构概览

```
用户上传文件
    │
    ▼
KnowledgeBaseController
    │
    ▼
KnowledgeBaseServiceImpl
    ├── 1. 读取文件文本（PDF/TXT/MD）
    ├── 2. 分块（滑动窗口：500 token/块，100 token 重叠）
    ├── 3. 向量化 → 写入 vector_store（PgVector，1024 维）
    └── 4. 写入 knowledge_bases（展示表，1 行/文件）
```

**两张表分工：**

| 表 | 用途 | 字段 |
|---|---|---|
| `knowledge_bases` | 用户展示表 | id, user_id, file_name, category, content(原文), created_at |
| `vector_store` | 检索表（PgVector） | id(uuid), content(分块), metadata(json), embedding(vector 1024) |

**设计原则：** `knowledge_bases` 轻量展示，`vector_store` 负责向量检索，两者通过 `metadata.fileName` + `metadata.userId` 关联。

---

### 2. API 接口

| 方法 | 路径 | 说明 |
|---|---|---|
| `POST` | `/api/knowledge-base/upload` | 上传文件，参数 `file` + `category`（专业知识问答/闲聊问答） |
| `GET` | `/api/knowledge-base/search` | 按文件名模糊搜索，`query` + `limit` |
| `GET` | `/api/knowledge-base/items` | 分页列表，`category` 筛选，`page` + `size` |
| `GET` | `/api/knowledge-base/file` | 查看文件原文，`fileName` |
| `PUT` | `/api/knowledge-base/file` | 编辑文件内容（重新向量化），`fileName` + `content` |
| `PUT` | `/api/knowledge-base/{id}/name` | 重命名（同时改 fileName + vector_store metadata） |
| `DELETE` | `/api/knowledge-base/file` | 删除文件（knowledge_bases + vector_store），`fileName` |
| `DELETE` | `/api/knowledge-base/clear` | 清空用户全部知识库 |

---

### 3. 检索方式

| 模式 | 方法 | 算法 | 参数 | 适用分类 |
|---|---|---|---|---|
| **语义检索** | `semanticSearch` | Cosine 相似度 | threshold=0.6, topK=4 | 闲聊问答 |
| **混合检索** | `hybridSearch` | 向量(0.7) + 全文 ts_rank(0.3) | topK=4 | 专业知识问答 |

语义检索直接从 `vector_store` 查询，按 `userId` + `category` 过滤，取 cosine≥0.6 的前 4 条。

混合检索在语义基础上叠加 PostgreSQL 全文检索权重，适合专业术语密集的内容。

---

### 4. RAG 问答流程（KnowledgeAgent）

```
用户选择知识库 + 提问
    │
    ▼
查询 knowledge_bases 获取该 KB 的 category
    │
    ├── 闲聊问答 → semanticSearch() → topK=4, threshold=0.6
    └── 专业知识问答 → hybridSearch() → topK=4
    │
    ▼
有结果 → 拼接上下文注入 LLM prompt → 流式回答
无结果 → 直接告知用户"知识库中暂无相关内容"
```

## 二、AI 记忆架构

### 1. 整体架构

```
用户对话
    │
    ▼
AgentRouter ──► DiaryAgent / PlanAgent / WeeklyReportAgent / KnowledgeAgent
    │
    ▼
AbstractAgent.buildContext(userId, input)
    ├── MemoryService.searchSimilarMemories()  → long_term_memories（长期，PgVector）
    └── MemoryService.getChatContext()         → Redis（短期，List）
    │
    ▼
LLM 回答（流式 SSE）
    │
    ▼
MemoryService.saveChatHistory()  → Redis + chat_histories
    │
    ▼
RabbitMQ 异步 → MemoryExtractAgent
    │
    ▼
MemoryService.saveMemory()  → long_term_memories（含 embedding）
    │
    ▼
[每周日 0:00] MemoryConsolidationScheduler
    │
    └── 全部记忆 → LLM 浓缩 → 1 条整合摘要
```

---

### 2. 短期记忆

**存储：** Redis List，key = `chat:context:{userId}`，上限 `MAX_SHORT_TERM` 条

**内容：** 最近对话的 role + content（JSON 格式）

**用途：** `buildContext()` 注入 LLM 上下文，提供多轮对话能力

---

### 3. 长期记忆

**存储：** PostgreSQL `long_term_memories` 表

| 字段 | 说明 |
|---|---|
| id | 主键 |
| user_id | 用户 ID |
| content | 记忆内容文本 |
| source_type | 来源类型（chat/diary/consolidation 等） |
| embedding | PgVector vector(1024)，用于相似度检索 |
| created_at | 创建时间 |

**记忆提取：** 每次 AI 对话完成后，RabbitMQ 异步触发 `MemoryExtractAgent`，从对话中提取关键信息并向量化存入。

**去重逻辑：**
- 先查相似记忆（cosine≥0.85）→ 有则合并更新
- 无则检查内容包含关系 → 重复则跳过
- 超出上限时淘汰最低相关度记忆

**记忆检索：** `searchSimilarMemories()` — 对查询文本生成 embedding，在 `long_term_memories` 中做向量相似度搜索。

---

### 4. 记忆整合定时任务

**组件：** `MemoryConsolidationScheduler`

**Cron：** `0 0 0 * * SUN`（每周日凌晨 0:00）

**流程：**
1. 获取所有用户 ID
2. 对每个用户，取出所有长期记忆（最多 200 条）
3. 拼接为编号列表，发送给 LLM
4. LLM 浓缩为 ≤500 字的用户画像摘要
5. 删除所有旧记忆
6. 写入 1 条整合后的 `sourceType="consolidation"` 记忆

**触发条件：** 用户记忆 ≥ 3 条才触发整合，少量记忆不处理。

---

### 5. Agent 体系

| Agent | 触发条件 | 职责 |
|---|---|---|
| **KnowledgeAgent** | 默认兜底 | 三模式对话：direct/rag/web |
| **DiaryAgent** | 含"日记/心情/回顾/记录" | 日记分析 |
| **PlanAgent** | 含"计划/目标/规划/学习路线" | 学习计划生成 |
| **WeeklyReportAgent** | 含"周报/总结/汇总/本周" | 周报生成 |
| **QueryRewriteAgent** | KnowledgeAgent 调用 | 模糊查询改写为精确检索词 |
| **MemoryExtractAgent** | RabbitMQ 异步 | 从对话提取长期记忆 |

路由逻辑：`AgentRouter` 按注册顺序遍历，`canHandle()` 匹配关键词，KnowledgeAgent 排最后作为兜底。

## 三、数据流示意

```
┌─────────────────────────────────────────────────────────┐
│                      用户输入                            │
└─────────────────────────────────────────────────────────┘
                           │
           ┌───────────────┼───────────────┐
           ▼               ▼               ▼
      DiaryAgent      PlanAgent      KnowledgeAgent
           │               │               │
           └───────────────┴───────┬───────┘
                                   │
                    ┌──────────────┴──────────────┐
                    │   AbstractAgent              │
                    │   buildContext()             │
                    │   ├─ 长期记忆 (PgVector)     │
                    │   └─ 短期对话 (Redis)        │
                    └──────────────┬──────────────┘
                                   │
                    ┌──────────────┴──────────────┐
                    │   LLM 流式回答 (SSE)         │
                    └──────────────┬──────────────┘
                                   │
              ┌────────────────────┴────────────────────┐
              ▼                                         ▼
     saveChatHistory()                    RabbitMQ → MemoryExtractAgent
     ├─ Redis (短期)                                   │
     └─ chat_histories (持久)                  saveMemory()
                                                   └─ long_term_memories

              ┌─────────────────────────────────────────┐
              │  MemoryConsolidationScheduler (每周日)   │
              │  全部记忆 → LLM浓缩 → 1条整合记忆        │
              └─────────────────────────────────────────┘
```

## 四、知识库检索架构

```
┌─────────────────────────────────────────────────────────┐
│                    问答页选择知识库                       │
│           闲聊问答 / 专业知识问答                         │
└────────────────────┬────────────────────────────────────┘
                     │
         ┌───────────┴───────────┐
         ▼                       ▼
   semanticSearch()        hybridSearch()
   cosine ≥ 0.6            向量0.7 + 全文0.3
   topK = 4                topK = 4
         │                       │
         └───────────┬───────────┘
                     ▼
              vector_store
         (userId + category 过滤)
                     │
         ┌───────────┴───────────┐
         ▼                       ▼
      有结果                   无结果
   拼接上下文注入LLM         直接告知用户
   "知识库中暂无相关内容"
```
